## Part 1: Epic 2 review

### Successes and strengths

1. **Epic 1 learnings applied deliberately** — Story specs explicitly carry forward **Team Agreements** (`th:href="@{...}"` only) and **process improvements** (validation + error-handling subtasks in task lists). Dev notes reference Epic 1 retro by name, which reduced repeat mistakes on new templates.

2. **Consistent vertical slices** — Each story delivered end-to-end: entity/repository/service/controller (where applicable), Thymeleaf UI, and tests. Package-by-feature layout (`com.examinai.course`, `com.examinai.task`, `com.examinai.review`) stayed clean.

3. **Schema-forward foundation paid off** — No new Liquibase churn for `course`, `task`, or `task_review` in Epic 2; stories correctly treat `001-init-schema.sql` as source of truth and prefer **DB-level cascade** over redundant JPA cascade.

4. **Security and transaction discipline held** — `@PreAuthorize` on individual methods, `@Transactional` on mutating service methods only, PRG on successful POSTs, POST+CSRF for deletes — aligned with architecture.

5. **Intern progress UX landed** — `TaskWithReview` record, `findForInternByUsername`, JOIN FETCH for `open-in-view: false`, and `fragments/task-status-card.html` satisfy UX-DR5/UX-DR11 and set up Epic 3 cleanly.

### Challenges and growth areas

1. **`open-in-view: false` forced reactive fixes** — Story 2.2 dev notes assumed LAZY + open-in-view; reality required **`@Transactional(readOnly = true)` on reads** and **JOIN FETCH** for list paths. Pattern: any new screen that walks associations in Thymeleaf needs an explicit fetch/transaction strategy in the same story, not as an afterthought.

2. **Admin vs mentor URL namespaces** — Hardcoded `/mentor/...` redirects and list links created awkward admin journeys until review patched **context-aware or parallel admin paths**. *Systemic fix:* define a small helper or model attribute for “current task/course base path” per request so templates stay DRY.

3. **Ownership and role rules surfaced late** — Mentor task edit/delete scope, mentor assignment validation, and `create()` exception handling were strengthened in **code review**, not first implementation. *Process tweak:* for CRUD stories, add an explicit subtask: “enumerate cross-user abuse cases (wrong mentor, stale IDs, non-MENTOR assignee).”

4. **Documentation vs code naming** — `epics.md` references `TaskService.findForIntern()`; implementation is **`findForInternByUsername(String)`**. Low risk but causes spec drift; align epic text or add a one-line alias in docs.

5. **Deferred backlog still carries schema/product debt** — `deferred-work.md` and story notes flag `task_review.status` typing, mentor FK delete behavior, `TaskReview.dateCreated` population contract for Epic 3, etc. Epic 3 should **triage and resolve or formally accept** these at kickoff.
